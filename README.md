# API-автотесты restful-booker (Postman)

Учебный проект по автоматизации API-тестирования в Postman. Реализована регрессионная коллекция для публичного REST API [restful-booker](https://restful-booker.herokuapp.com) с полным CRUD-циклом, авторизацией и негативными сценариями.

## Что тестируется

REST API сервиса бронирования: создание, чтение, обновление и удаление броней, авторизация по токену, обработка некорректных запросов.

## Структура коллекции

Коллекция «Регрессионное тестирование» разбита на 4 папки (30 проверок):

### 01 Подготовка
- **Получение токена** — POST /auth, авторизация, сохранение токена в переменную
- **Создание тестовых данных** — POST /booking, эталонная бронь с фиксированными данными
- **Получение эталонной брони** — GET /booking/{id}, проверка структуры ответа
- **Получение всех броней** — GET /booking, проверка массива и итерация по элементам

### 02 Основные сценарии (CRUD)
- **Создание пользователя** — POST /booking, генерация случайных данных, проверка совпадения
- **Обновление пользователя** — PUT /booking/{id}, обновление с авторизацией
- **Удаление пользователя** — DELETE /booking/{id}, удаление с авторизацией

### 03 Проверка ошибок (негативные тесты)
- **Отсутствие прав** — PUT без токена → ожидается 403 Forbidden
- **Запрос с неверными данными** — неполное тело → ожидается ошибка
- **Проверка пустых полей** — пустой объект → ожидается ошибка

### 04 Очистка
- **Удаление тестовых записей** — удаление созданных данных (teardown)

## Применённые техники и инструменты

- HTTP-методы: GET, POST, PUT, DELETE
- Авторизация по токену (получение и передача в заголовке)
- Переменные окружения, передача данных между запросами
- Pre-request скрипты — генерация тестовых данных
- Tests (ассерты): проверка статус-кодов, свойств, типов данных, массивов, вложенных объектов
- Позитивные и негативные сценарии
- Обработка исключений (try-catch), итерация по списку (forEach)
- Collection Runner — запуск регрессионного набора

## Как запустить

1. Импортировать в Postman оба файла:
   - `restful-booker-tests.postman_collection.json` (коллекция)
   - `Restful-booker.postman_environment.json` (окружение)
2. Активировать окружение **Restful-booker**.
3. Запустить коллекцию целиком через **Collection Runner** (правый клик на коллекции → Run).

Токен авторизации получается автоматически в первом запросе. Учётные данные тестового стенда: `admin` / `password123`.

## Особенности тестового стенда

restful-booker — публичный учебный API. Данные на нём нестабильны (брони периодически перезатираются), поэтому проверки чтения построены на валидации структуры ответа, а полный прогон выполняется атомарно через Collection Runner.
